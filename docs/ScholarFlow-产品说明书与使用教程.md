# ScholarFlow 产品说明书与使用教程

> **版本**: 1.0 | **最后更新**: 2026-05-08
>
> ScholarFlow 是一个 AI 驱动的学术研究全流程平台，覆盖从文献检索到 Research Proposal 生成的完整科研工作流。平台集成 DeepSeek、Gemini、GPT-4o、Claude 等多个大语言模型，对接 Google Scholar、Semantic Scholar、OpenAlex、arXiv 等 11 个学术数据源，为管理学及相关学科的研究者提供一站式智能科研体验。

### 一、系统架构总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Next.js 16 (App Router) + React 19 + TypeScript | Turbopack 构建，SSR + 客户端混合渲染 |

| 分析引擎 | Stanford STORM（内置）、Google NotebookLM（外接 MCP） | 多视角分析 + 源引用追溯 |

**数据流全景**：

```
用户输入查询
    ↓
① 文献检索 → AI 关键词提取 + 8 维同义词扩展 → 11 源并行检索 → 三级优先级排序 → AI 相关性评分
    ↓
② 文献库 → PDF 上传 + 全文提取 → 批量 AI 深度分析（标签、变量、方法、贡献）
    ↓
③ 文献综述 → STORM 多视角大纲 → 并行章节生成 → 连贯性润色 → 流式输出
    ↓
④ 综述优化 → Word 初稿上传 → 结构分析 → 缺口检索 → 双轨修改计划 → AI 重写
    ↓
⑤ 知识图谱 → 并行子代理编码 → 变量关系图谱 → 元分析证据表 → 全景报告
    ↓
⑥ 研究想法 → 理论×情境×方法矩阵 → 新颖性评分 → 模拟同行评审
    ↓
⑦ 理论整合 → 识别理论框架 → 跨理论连接 → 整合框架
    ↓
⑧ 概念模型 → AI 自动提取变量关系 → 可视化拖拽画布 → 假设标注
    ↓
⑨ Research Proposal → 结构化 7 章节生成 → APA 引用 → 流式输出
    ↓
⑩ 参考文献 → APA / MLA / Chicago / GB/T 7714 / BibTeX 批量导出
```

### 二、各功能模块详解

---

#### 2.1 智能文献检索

**功能定位**：从自然语言输入出发，自动生成专业学术检索策略，跨 11 个数据源并行检索，经 AI 相关性评分和期刊质量分级后返回高质量论文列表。

##### 技术实现

**Step 1 — AI 关键词提取与同义词扩展**

系统首先将用户的自然语言输入（支持中英文混合）发送给 DeepSeek V4 Flash，执行 6 步提取流程：

1. **翻译**：将中文输入翻译为英文学术表达（去除"帮我找""有关"等口语化词汇）
2. **意图分类**：判断查询类型 —— 主题型（TOPICAL）、关系型（RELATIONAL）、方法型（METHODOLOGICAL）、综述型（REVIEW）
3. **关键词提取**（3-6 个）：提取完整学术短语，不拆分复合术语（如 "AI washing" 保持为一个整体）
4. **8 维同义词扩展**（每词 10-15 个）：

| 维度 | 说明 | 示例 ("AI sycophancy") |
|------|------|----------------------|
| ① 直接同义词 | 完全相同概念 | AI agreeableness, AI flattery |
| ② 上位概念 | 更广泛的类目 | AI alignment failure, value misalignment |
| ③ 下位子类 | 具体表现形式 | reward hacking, specification gaming |
| ④ 邻接研究流 | 不同学科的同一现象 | RLHF failure modes, human feedback bias |
| ⑤ 正式学术术语 | Nature/Science 论文用语 | sycophancy, sycophantic behavior |
| ⑥ 因果机制 | 成因与后果 | reward model overoptimization |
| ⑦ 测量方法 | 研究者如何测量 | sycophancy benchmark, opinion conformity test |
| ⑧ 历史术语 | 概念的旧称 | yes-man AI, social desirability bias in AI |

5. **构建查询**：精确查询（5-8 条，每个关键词 + 顶级同义词作精确短语）+ 宽泛查询（3-5 条，OR 连接同义词组，覆盖不同维度组合）
6. **过滤器提取**：从输入中分离质量要求（如 "ABS 3 星以上" → `minABS: "3"`）和时间范围（如 "近三年" → `yearFrom: 2023`）

**学术策略 — 双向解读**：对于关系型查询（如 "AI 与碳排放"），系统强制生成覆盖**两个因果方向**的关键词 —— 既包括 "AI carbon footprint"（AI 产生碳排放），也包括 "AI for decarbonization"（AI 帮助减排）。遗漏任一方向即丢失约一半文献。

**Step 2 — 多源并行检索**

系统同时向以下数据源发起检索请求：

| 数据源 | 覆盖范围 | 特点 |
|--------|---------|------|
| Google Scholar (via Serper) | 最广泛 | 需合并精确查询以减少 API 调用 |
| Semantic Scholar | 2 亿+ 篇 | 提供 TLDR 摘要、引文上下文 |
| OpenAlex | 2.5 亿+ 篇 | 免费、倒排索引摘要重建 |
| arXiv | 预印本 | 最新前沿 |
| CORE | 2 亿+ OA 论文 | 英国开放获取聚合 |
| CrossRef | DOI 元数据 | 摘要 + 引用量 |
| Unpaywall | OA 定位 | 找到论文的免费全文链接 |
| PubMed / DBLP / bioRxiv | 专业领域 | 生物医学 / 计算机 |
| Web of Science | 核心合集 | 需机构订阅 |

检索超时控制：Google Scholar 查询 30 秒超时，免费源 3-5 条查询并行（30 秒超时），确保 SSE 流不中断。

**Step 3 — 去重与摘要补全**

去重策略：按 DOI 归一化合并（优先），若无 DOI 则按标题关键词匹配。

4 轮摘要补全管线（并行执行）：
1. CrossRef：DOI → 完整摘要
2. Semantic Scholar：DOI 或标题搜索 → 摘要 + TLDR
3. OpenAlex：标题搜索 → 从倒排索引重建摘要
4. Springer HTML：仅 Springer DOI → 直接 HTML 抓取

第 5 轮（兜底）：对仍无摘要的论文，启用全文获取器（见 2.2）尝试获取前 2000 字。

**Step 4 — 三级优先级排序**

| 优先级 | 条件 | 说明 |
|--------|------|------|
| P1（最高） | Nature/Science 系列、ABS 3*+、JCR Q1、UTD24、FT50 | 不看引用量，永远最前 |
| P2（次高） | 年份动态阈值：2026 年 >0 引、2025 年 >30 引、2024 年 >100 引、≤2023 年 >200 引 | 高引经典 |
| P3（第三） | 2026 年 arXiv 且引用 >0 | 最新前沿 |

在每个优先级内部，按 AI 相关性评分 → 期刊等级 → 影响因子 → 引用量排序。

**Step 5 — AI 相关性评分**

对每篇论文发起独立 LLM 调用（80 并发），基于标题 + 摘要 + S2 引文上下文（被引论文中引用该论文的原文句子），打出 0-10 分并附中文理由。

评分维度：关键词匹配度、学术贡献、方法论质量、创新性。阈值：limit=20 时保留 ≥6 分，limit=100 时保留 ≥5 分。

#### 2.2 文献库管理

**功能定位**：统一管理检索入库的论文和用户手动上传的 PDF，提供批量 AI 深度分析、全文提取、Zotero 双向同步等能力。

##### 技术实现

**PDF 上传与全文提取**

- 支持单文件上传、文件夹批量上传（5 并发）、从 Zotero 导入
- 文本提取使用 `unpdf`（基于 pdf.js，~150ms/文件），无需外部服务
- 提取后自动识别摘要（正则匹配 Abstract/摘要 标志），存储全文至数据库
- PDF 文件存储于阿里云 OSS（香港），通过签名 URL 提供下载

**全文获取器（8 级瀑布策略）**

对于通过检索入库但无 PDF 的论文，系统自动尝试 8 种策略获取全文：

| 层级 | 策略 | 超时 | 适用场景 |
|------|------|------|---------|
| Tier 1 | Europe PMC + S2 + CORE | 6 秒竞速 | 最快的 3 个 OA 源 |
| Tier 2 | PMC + Unpaywall + CrossRef TDM + Springer + OpenAlex + 出版商 HTML | 8 秒竞速 | 覆盖更广 |
| Tier 3 | GROBID（ML PDF 解析）+ EuropePMC 标题搜索 | 每策略 5 秒 | 最后兜底 |

竞速策略：Tier 1/2 内所有策略并行发起，第一个返回 ≥200 字结果即胜出，其余立即放弃。

出版商特定 HTML 提取器：针对 Wiley、Elsevier、Nature/Springer、T&F、SAGE、AAAS 分别编写 CSS 选择器，直接从落地页提取正文（即使付费论文，落地页也通常展示大量内容）。

**批量 AI 深度分析**

对有全文/摘要的论文（100 并发 LLM 调用），自动提取：
- 研究标签（3-5 个学术关键词）
- 理论模型（使用了什么理论）
- 核心变量（自变量 / 因变量 / 中介 / 调节 / 控制）
- 研究方法（实证 / 实验 / 案例 / 元分析等）
- 主要贡献（1-2 句中文摘要）

**期刊排名系统**

系统内置多维度期刊数据库：

| 排名体系 | 数据量 | 分级 |
|----------|--------|------|
| ABS 学术期刊指南 | 1,635 种 | 1 / 2 / 3 / 4 / 4* |
| JCR（Web of Science） | 全量 | Q1-Q4 |
| SSCI / SCI | 标记 | 是/否 |
| UTD24 | 24 种 | 是/否 |
| FT50 | 50 种 | 是/否 |
| 中科院分区 | - | 一区-四区 |
| CCF（计算机） | - | A / B / C |
| CSSCI / 北大核心 | - | 是/否 |

论文列表中以彩色徽章展示排名：`UTD24` `FT50` `ABS 4*` `JCR Q1` `SSCI` 等。

#### 2.3 文献综述生成

**功能定位**：基于项目文献库中已上传全文的论文，使用 STORM 多视角框架自动生成结构化文献综述初稿。

##### 技术实现

**Phase 1 — 多视角大纲生成**（1 次 LLM 调用）

输入前 30 篇论文的标题、摘要（200 字截断）、期刊等级，生成包含以下内容的结构化大纲：
- 综述标题
- 研究视角（默认 4 个：理论视角、实证方法视角、应用情境视角、批评与争议视角）
- 章节列表（每章含标题、所属视角、核心发现、引用文献编号）
- 研究空白（3-5 条）
- 未来方向（3-5 条）

**Phase 2 — 并行章节撰写**（最多 4 个章节并行）

每个章节独立生成：
- 输入：章节标题、视角、核心发现列表、相关论文的摘要（400 字）+ 全文节选（3,000 字）
- 每章目标字数 = 总目标字数 × 85% ÷ 章节数
- 要求：每个论点标注引文编号 `[1]`、标注期刊等级（UTD24/FT50/ABS4*）

**Phase 3 — 连贯性润色**（1 次流式 LLM 调用）

将所有章节拼接后，通过流式生成进行最终润色：
- 添加引言段（综述目的、范围、结构说明）
- 添加总结段（研究空白汇总、未来方向）
- 确保章节间逻辑过渡自然
- 保留所有引文标注
- 附完整参考文献列表

**学术策略**：STORM（Synthesis of Topic Outlines through Retrieval and Multi-perspective questioning）框架的核心理念是从**多个研究视角**而非单一线性叙述来组织综述。这样能更全面地覆盖不同研究社区的观点，避免单一视角偏见。

#### 2.4 综述优化

**功能定位**：用户上传已有的 Word 文献综述初稿，系统自动分析结构、识别覆盖缺口、检索补充文献、生成双轨修改计划、执行 AI 优化重写。

##### 技术实现

**5 阶段工作流**：

**阶段 1 — 初稿分析**（1 次 LLM 调用）
- 提取研究主题、关键主题（3-8 个）、已引参考文献列表
- 分析各章节结构（标题 + 摘要 + 引用量）
- 识别薄弱章节（引用不足、论证空白）
- 生成 5-10 个英文学术搜索关键词

**阶段 2 — 缺口文献检索**
- 使用阶段 1 生成的关键词，调用与文献检索**完全相同的 `smartSearch()` 函数**（limit=50，启用 AI 评分）
- 对检索到的论文进行 20 并发逐篇 AI 分析（核心发现、方法、贡献）
- **重要**：综述优化中的文献检索与独立的文献检索功能使用完全相同的底层管线，确保结果一致性

**阶段 3 — 缺口分析**（1 次 LLM 调用）
- 交叉比对初稿分析 + 新发现论文 + 文献库论文
- 输出覆盖缺口（4-8 个，每个含主题、严重程度、推荐论文索引）
- 输出薄弱章节改进建议 + 文献库中未引用的相关论文

**阶段 4 — 双轨修改计划**（1 次 LLM 调用）

| 轨道 | 标识 | 说明 |
|------|------|------|
| 已有内容改进 (Improve) | 【改进】 | 加深现有章节的理论分析、补充引用、修复论证空白 |
| 方向扩展延伸 (Extend) | 【新方向】 | 添加初稿未覆盖的新视角、新趋势、批判性观点 |

每个修改项必须指定具体使用哪些论文（精确标题匹配）。

**阶段 5 — AI 优化重写**（1 次流式 LLM 调用）
- 输入：原始初稿（12,000 字截断）+ 修改计划 + 文献库论文全文 + 检索论文摘要
- 引用格式：全部使用 APA 行内引用 `(Author, Year)`
- 改进段落标记 【改进】，新增段落标记 【新方向】
- 附完整更新后参考文献列表

**学术策略**：双轨修改计划的设计灵感来自期刊审稿意见的常见结构 —— 审稿人通常分别提出 "现有内容的改进建议" 和 "建议探索的新方向"。将两者分开有助于作者区分修补型修改和扩展型修改。

#### 2.5 知识图谱

**功能定位**：从文献库全文中提取所有变量间因果关系，生成交互式知识图谱、元分析证据表、领域全景报告。

##### 技术实现

**并行子代理编码架构**

- 将论文分为每组 8 篇，启动最多 8 个并行子代理
- 每个子代理作为"元分析编码员"，从分配的论文中提取：
  - **节点**（变量）：ID、类型（自变量/因变量/中介/调节/控制）、出现频次、同义变量列表、测量方式
  - **边**（关系）：源变量 → 目标变量、方向（正/负/混合/不显著）、权重、逐篇发现（效应量 β、样本、方法、年份）、一致性评估、边界条件、证据强度

**图谱合并策略**
- 节点去重：按归一化 ID 匹配（处理 "企业绩效" = "公司绩效" = "organizational performance" 等同义情况）
- 边合并：相同源→目标变量对的边合并发现、叠加权重、重新评估一致性

**元分析综合**（2 次并行 LLM 调用）
1. **元摘要**：领域名称、核心发现（含 k 值和一致性）、理论版图、方法学画像、研究空白、成熟度评估（🌱新兴 → 📈成长 → 🌳成熟 → 🏛️饱和）
2. **全景报告**：1,500+ 字叙事性分析报告，涵盖概述、核心变量、一致性评估、理论框架、方法论、研究空白、未来议程

**可视化**：基于 React Flow 的力导向图，节点按变量类型着色，边标注方向（+/-/±/n.s.），支持点击查看逐篇证据。

**学术策略**：本模块模拟了系统综述中的"编码"（coding）环节 —— 每个子代理相当于一位独立编码员，各自提取变量关系后合并。这种设计既能并行加速处理 40-50 篇文献，又能通过合并步骤处理编码不一致的情况。

#### 2.6 研究想法生成

**功能定位**：基于文献分析，生成创新性研究想法并模拟同行评审。

##### 技术实现

**6 步管线**：

1. **维度提取**（1 次 LLM 调用）：从 20 篇论文中提取三个创新维度
   - 理论维度（4-8 个已有理论）
   - 情境维度（4-8 个研究情境）
   - 方法维度（4-8 种研究方法）
   - 研究空白（≥3 个）

2. **想法生成与排名**（1 次 LLM 调用）：
   - 在理论 × 情境 × 方法的三维矩阵中寻找新颖组合
   - 优先填补已识别的研究空白
   - 生成 5-8 个想法，每个包含标题、假设、贡献
   - 三维评分（每项 0-10）：新颖性（×0.4）+ 可行性（×0.3）+ 影响力（×0.3）

3. **模拟同行评审**（3 次并行 LLM 调用）：
   - 对排名前 3 的想法，模拟顶级管理学期刊（AMJ, ASQ, SMJ）匿名审稿人
   - 输出：优点、不足、需要回答的问题、评审意见（strong_accept / accept / revise / reject）

**学术策略**：维度提取 → 组合生成的范式借鉴了计算创造力（computational creativity）中的"概念融合"（conceptual blending）方法。通过显式化理论×情境×方法的创新空间，避免 AI 仅生成表面相似的想法，而是产出结构性创新的研究方案。

#### 2.7 理论整合

**功能定位**：识别文献中使用的所有理论框架，发现跨理论连接，生成整合性框架。

##### 技术实现

（1 次 LLM 调用，输入 20 篇论文摘要 + 研究主题）

**输出结构**：
- **理论列表**：名称（中英文）、核心构念、关键假设、边界条件、引用文献编号
- **跨理论连接**：理论 A ↔ 理论 B 的共享构念、整合潜力、连接强度（strong/moderate/weak）
- **整合框架**：框架标题、描述、中心理论、多层结构（每层含名称、理论、角色）

**可视化**：左侧显示整合框架卡片（多层结构）+ 跨理论连接卡片，右侧显示各理论详情侧栏。

**学术策略**：理论整合是管理学研究中的高阶任务，通常出现在综述论文的理论框架章节。本模块帮助研究者快速识别文献中隐含的理论关联（如拟人化理论与心智感知理论共享"类人感知"构念），为构建跨理论整合框架提供结构化起点。

---

#### 2.8 概念模型

**功能定位**：从文献中自动提取变量关系并生成可视化概念模型，支持拖拽编辑。

##### 技术实现

**AI 自动生成**：调用 `/api/ai/analyze` 的 `variables` 类型

**批量处理策略**（>10 篇论文时自动启用）：
- 将论文分为每组 10 篇，4 组并行 LLM 调用
- 每组内论文重新编号 [1]-[10]，提取变量关系
- 合并时将编号映射回全局序号
- 去重：相同自变量→因变量对合并，整合来源、中介、调节变量
- 方向冲突处理：不同批次发现不同方向时标记为 "mixed"

**提取格式**（每条关系）：
```json
{
  "independentVar": "AI 拟人化程度",
  "dependentVar": "消费者信任",
  "mediators": ["感知温暖"],
  "moderators": ["产品类型"],
  "direction": "positive",
  "effectSize": "β = 0.42",
  "sampleContext": "中国消费者，在线购物情境",
  "sources": [1, 5, 12]
}
```

**可视化画布**：基于 React Flow，支持：
- 按类型着色节点（自变量绿色、因变量蓝色、中介变量琥珀色、调节变量紫色、控制变量灰色）
- 从节点圆点拖拽创建假设连线（自动标注 H1, H2, H3...）
- 双击编辑节点名称和假设标签
- 手动添加任意类型的变量节点

---

#### 2.9 Research Proposal 生成

**功能定位**：基于文献库论文和研究想法，生成结构化学术研究计划书。

##### 技术实现

（1 次流式 LLM 调用）

**输入准备**：
- 研究主题（必填）
- 论文列表（预格式化 APA 引用：Authors, Year, Title, Journal）
- 研究想法（可选，从想法模块跨页面传递）

**输出结构（7 章节）**：

| 章节 | 内容 | 目标字数 |
|------|------|---------|
| 1. 研究标题 | 中英文学术化标题 | 1-2 句 |
| 2. 引言 | 研究背景、问题提出、意义、创新点 | 200-400 字 |
| 3. 文献综述 | 按主题聚类归纳、标注 [Author, Year]、研究空白 | 200-400 字 |
| 4. 理论框架与假设 | 核心理论、概念模型、H1/H2/H3 推导逻辑 | 200-400 字 |
| 5. 研究方法 | 设计、样本、变量测量、分析方法 | 200-400 字 |
| 6. 预期贡献 | 理论贡献、实践意义 | 200-400 字 |
| 7. 参考文献 | APA 格式完整引用 | - |

**流式输出**：逐 token 生成，左侧显示章节导航栏可快速跳转，右侧实时渲染 Markdown。

#### 2.10 参考文献导出

**功能定位**：从文献目录中批量生成各格式参考文献列表。

##### 技术实现

支持 5 种格式：

| 格式 | 适用场景 |
|------|---------|
| APA 7th | 管理学、心理学、社科（默认） |
| MLA 9th | 人文学科 |
| Chicago | 历史学、部分社科 |
| GB/T 7714 | 中文论文 |
| BibTeX | LaTeX 用户 |

格式化引擎：优先通过 `citation-js` + DOI 查询获取出版商标准元数据，若失败则回退到本地格式化器（根据作者数量自动处理 et al. 规则）。

支持全选/手动勾选论文，一键复制全部引用文本。

## 第二部分：使用教程

---

### 教程一：快速开始

#### 1. 注册与登录

1. 访问 ScholarFlow 首页，点击 **"创建研究项目"**
2. 在登录页点击 **"注册"**，填写姓名、邮箱、密码
3. 提交后进入 **审核等待** 状态，管理员审核通过后会发送邀请码
4. 收到邀请码后，回到登录页选择 **"已有邀请码"**，输入邮箱和 8 位邀请码完成注册
5. 使用邮箱和密码登录

#### 2. 创建研究项目

1. 登录后进入项目列表页，点击 **"+ 创建新项目"**
2. 填写：
   - **项目名称**（必填）：如 "AI 拟人化与消费者信任"
   - **研究领域**（选填）：如 "营销管理、消费者行为"
   - **项目描述**（选填）：研究目标简述
3. 点击 **"创建项目"**，进入项目概览页

#### 3. 理解项目概览

项目概览页显示 4 个统计卡片（文献总数、已上传原文、研究想法、图谱节点）和 **快速开始 6 步流程**。按顺序完成即可走通全流程。

---

### 教程二：文献检索与入库

#### Step 1：智能检索

1. 点击侧栏 **① 文献检索**
2. 在搜索框输入研究主题（支持中英文混合），例如：
   - `"AI anthropomorphism and consumer trust"`
   - `"AI拟人化对消费者信任的影响，ABS 3星以上，近三年"`
3. 点击 **"智能检索"**，系统自动执行：
   - AI 提取关键词 → 扩展同义词 → 构建精确/宽泛查询
   - 跨 11 个数据源并行检索
   - AI 逐篇相关性评分 + 期刊质量分级排序
4. 结果列表中每篇论文显示：
   - 标题、作者、年份、期刊
   - 期刊排名徽章（UTD24、FT50、ABS 4*、JCR Q1 等）
   - 引用量、AI 相关性评分（0-10）
5. 勾选需要的论文，点击 **"加入文献库"** 导入项目

> **提示**：如需搜索中文文献（CNKI），在搜索框旁切换至 **"中文期刊"** 模式。

#### Step 2：上传 PDF 全文

1. 进入 **② 文献库**
2. 点击 **"上传 PDF"** 或 **"批量导入文件夹"**
3. 系统自动提取全文 + 识别元数据（标题、摘要、作者）
4. 上传完成后，论文旁显示 **"查看原文"** 按钮

> **重要**：后续的综述生成、知识图谱、研究想法等功能**仅使用有全文的论文**。建议至少上传 20-40 篇 PDF。

#### Step 3：批量 AI 分析

1. 在文献库页面，点击 **"AI 综合分析"** 按钮
2. 系统对所有论文进行 AI 深度分析（100 并发），提取标签、变量、方法、贡献
3. 分析完成后，展开任意论文可查看 AI 分析结果

---

### 教程三：生成文献综述

#### 方式 A：从零生成

1. 点击侧栏 **③ 文献综述** → **"综述初稿生成"**
2. 输入研究主题，如 "AI 拟人化对消费者行为的影响"
3. 设置目标字数范围（默认 6000-8000 字）
4. 选择分析引擎（默认 STORM）和 AI 模型
5. 点击 **"生成综述"**
6. 观察三阶段进度：大纲生成 → 章节撰写 → 连贯润色
7. 生成完成后：
   - 左侧：查看结构化大纲（章节 + 视角 + 研究空白 + 未来方向）
   - 右侧：编辑完整综述文本

#### 方式 B：优化已有初稿

1. 点击 **③ 文献综述** → **"综述优化"**
2. 上传 Word 格式（.docx）的文献综述初稿
3. 点击 **"分析初稿"**，查看 AI 识别的：
   - 研究主题、关键主题、章节结构
   - 薄弱章节（标红提示）
   - 推荐搜索关键词
4. 在关键词面板中选择/取消/自定义搜索方向
5. 点击 **"检索补充文献"**，系统自动搜索并逐篇分析
6. 查看覆盖缺口报告，勾选需要纳入的论文到 **修改篮**
7. 点击 **"生成修改计划"**，查看双轨计划（改进 + 新方向）
8. 确认后点击 **"执行优化"**，AI 重写综述
9. 在下方 AI 对话框中可继续微调

---

### 教程四：构建知识图谱

1. 点击侧栏 **④ 知识图谱**
2. 确认已有足够全文论文（建议 20+）
3. 点击 **"生成领域全景分析"**
4. 等待并行子代理完成编码（约 3 分钟）
5. 查看三个标签页：
   - **变量关系图谱**：交互式力导向图，点击节点/边查看详情
   - **元分析证据表**：核心发现、关系证据表（含 k 值、一致性）、理论版图、方法学画像、研究空白
   - **领域全景报告**：1,500+ 字叙事性分析
6. 在页面底部使用 AI 对话框，针对图谱数据提问

---

### 教程五：生成研究想法

1. 点击侧栏 **⑤ 研究想法**
2. 点击 **"生成研究想法"**
3. 查看维度提取结果（理论 / 情境 / 方法三张卡片 + 研究空白列表）
4. 查看生成的 5-8 个想法，按综合评分排序：
   - 每个想法显示标题、理论/情境/方法标签、评分（0-10）
   - 展开查看核心假设、预期贡献、评分柱状图
   - 前 3 名含模拟同行评审（优点/不足/问题/评审意见）
5. 对感兴趣的想法：
   - 点击 **"搜索相关文献"** → 跳转文献检索页自动填入主题
   - 点击 **"推送到 Obsidian"** → 导出到笔记软件
   - 点击 **"复制"** → 用于进一步研究

---

### 教程六：理论整合与概念模型

#### 理论整合

1. 点击侧栏 **⑥ 理论整合**
2. 可选输入聚焦主题
3. 点击 **"分析理论"**
4. 查看结果：
   - 左侧：整合框架（多层结构）+ 跨理论连接（共享构念、整合潜力）
   - 右侧：各理论详情（核心构念、假设、边界条件）

#### 概念模型

1. 点击侧栏 **⑦ 概念模型**
2. 两种方式：
   - **AI 自动生成**：点击 "AI 自动生成模型"，系统从全文中提取变量关系
   - **手动构建**：点击工具栏中的 "+ 自变量" "+ 因变量" 等按钮添加节点
3. 在画布上拖拽节点布局，从节点圆点拖出连线创建假设
4. 双击编辑节点名称或假设标签（H1, H2...）

### 教程七：生成 Research Proposal

1. 点击侧栏 **⑧ Proposal**
2. 填写 **研究主题**（必填），如 "AI 拟人化对消费者信任的影响研究"
3. （可选）粘贴研究想法模块的想法到 "研究想法" 文本框
4. 设置目标字数
5. 点击 **"生成 Proposal"**
6. 实时查看 7 个章节逐步生成（左侧导航栏可跳转）
7. 生成完成后：
   - 点击 **"导出 Markdown"** 保存
   - 点击 **"复制全文"** 到剪贴板
   - 点击 **"导出参考文献 (APA)"** 单独导出引用列表

### 教程八：导出参考文献

1. 点击侧栏 **⑨ 参考文献**
2. 选择格式：APA 7th / MLA 9th / Chicago / GB/T 7714 / BibTeX
3. 勾选需要导出的论文（或点击 "全选"）
4. 点击 **"生成参考文献"**
5. 点击 **"复制全部"** 粘贴到论文中

### 最佳实践建议

1. **文献量**：建议项目中至少有 **20-40 篇已上传全文的论文**，这是综述生成、知识图谱等功能的最佳输入量
2. **工作流顺序**：推荐按 检索 → 上传 PDF → 综述 → 图谱 → 想法 → 理论 → 模型 → Proposal 的顺序逐步深入
3. **AI 模型选择**：
   - 日常使用选 **DeepSeek V4 Flash**（最快）
   - 重要分析选 **DeepSeek V4 Pro**（最强）
   - 深度推理选 **DeepSeek R1**
4. **综述优化**：先手动写一版初稿（哪怕很粗糙），再用综述优化功能补充 —— 效果远好于从零生成
5. **跨功能联动**：知识图谱的"研究空白"可直接导入到研究想法模块；研究想法可直接传递到 Proposal 模块
