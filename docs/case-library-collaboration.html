Phase 4: 性能危机 — usePersistedState

85 个 hooks 同时解析 JSON，导致 Mac 鼠标失灵

May 11 系统级 Bug

性能危机 浏览器卡死

User
打开网页还是会卡，Mac 经常鼠标失灵
Root Cause 85 个 usePersistedState hooks 在组件挂载时同时从 sessionStorage 读取和解析 JSON，阻塞主线程
Fix 将 sessionStorage 读取包裹在 requestIdleCallback 中，延迟到浏览器空闲时执行，结合 prefetch={false} 减少预加载
// Before: 阻塞主线程 const stored = sessionStorage.getItem(key); const parsed = JSON.parse(stored); // x85 同时执行 // After: 延迟到空闲时 requestIdleCallback(() => { const stored = sessionStorage.getItem(key); if (stored) setState(JSON.parse(stored)); });
UI Freeze requestIdleCallback prefetch={false}

协作模式复盘

4 个会话中浮现的角色分工、反复出现的模式与关键决策

User 的角色

  • 产品定义者 — 统一身份、两栏布局、三引擎架构等方向性决策
  • 质量哨兵 — 发现案例不更新、检索不全面、评分不一致等真实场景 Bug
  • 策略指引者 — 集成理论知识库、移植想法模块等高层策略
  • 体验守护者 — 检索结果要有序、鼠标不能卡、速度要快

Claude 的角色

  • 架构实现者 — 数据模型、API 路由、前端组件、SSE 管道
  • 根因诊断者 — AND/OR 逻辑、双重过滤、字段名不匹配、85 hooks 并发
  • 知识工程师 — 35+ OB 理论知识库、期刊列表官方校验
  • 性能调优者 — 并发控制、token 裁剪、idle callback、预截断

反复出现的模式

  • 覆盖率 vs 速度拉锯 — 检索管道是跨会话最频繁调试的组件：扩大覆盖 → 超时/限流 → 缩小覆盖 → 漏论文 → 再扩大
  • 级联 Bug — 修一个 Bug 引入另一个（预截断修复 → 丢弃小众论文 → 0 结果）
  • 字段名不一致 — 管道和前端之间的命名不匹配（contribution vs relevanceContribution）

关键架构决策

  • 三引擎并存 — 从用户追问自然演化，非预设方案
  • GROBID 替代 unpdf — Docker 本地部署的 ML 解析器，质量远超规则解析
  • 理论知识库分层筛选 — 全量知识库用于分析，按 topic 裁剪版用于快速生成
  • 质量源优先排序 — ABS 3+、JCR Q1、arXiv 在预截断中优先保留
